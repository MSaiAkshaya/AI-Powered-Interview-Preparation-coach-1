# AI-Powered Interview Prep Coach — IBM Cloud Edition

A Streamlit app that generates tailored mock-interview questions from your
resume + a job description, lets you type your answers, analyzes them, and
builds a skill-gap learning roadmap — powered entirely by **IBM Cloud**
services.

> Built for the **AICTE IBM SkillsBuild Internship**.

## Features

- Resume (PDF) + Job Description + Target Company input
- Interview type selector: Behavioral / Technical / HR
- Difficulty level: Beginner / Intermediate / Advanced
- Company-flavored question generation (uses the model's general knowledge)
- Type your answer to each question directly in the app
- Per-answer AI feedback: relevance score, STAR structure check, filler-word
  flag, pacing note, strengths, improvement suggestion — generated by the
  **IBM Granite Foundation Model** on **IBM watsonx.ai**
- Resume-to-Roadmap tab: skill gap analysis with a study plan per missing skill

## IBM Cloud Services Used

| Service | Role in this project |
|---|---|
| **IBM watsonx.ai** | Hosts and serves the foundation model used for every text-generation call |
| **IBM Granite Foundation Model** (`ibm/granite-3-8b-instruct` by default) | Generates interview questions, analyzes answers, and builds the skill-gap roadmap |
| **IBM Cloud Project** (watsonx.ai project) | Scopes and authorizes every inference request via `project_id` |
| **IBM Cloud API Key (IAM)** | Authenticates every request to watsonx.ai |

## Tech Stack

| Layer | Tool |
|---|---|
| UI | Streamlit |
| LLM | IBM watsonx.ai — IBM Granite Foundation Model |
| PDF parsing | pdfplumber |
| Config / auth | python-dotenv, IBM Cloud API Key (IAM) |

## SETUP — Run These Steps In Order

### Step 1: Get the code on your machine
If you already have this folder locally, skip to Step 2.

### Step 2: Create a virtual environment (recommended, keeps installs isolated)
```bash
python -m venv venv
```
Activate it:
```bash
# Windows
venv\Scripts\activate

# Mac/Linux
source venv/bin/activate
```
You'll know it worked because your terminal prompt now shows `(venv)` at the start.

### Step 3: Install all dependencies
```bash
pip install -r requirements.txt
```

### Step 4: Set up IBM Cloud + watsonx.ai

1. **Create an IBM Cloud account** (free tier is enough): https://cloud.ibm.com/registration
2. **Create an IBM Cloud API Key**
   - Go to https://cloud.ibm.com/iam/apikeys → **Create an IBM Cloud API key**
   - Name it (e.g. `interview-coach-key`) and copy the key immediately — it's
     only shown once.
3. **Create a watsonx.ai project (your "IBM Cloud Project")**
   - Go to https://dataplatform.cloud.ibm.com/wx/home
   - Create a new **Project** (or use an existing one)
   - Open **Project → Manage → General** and copy the **Project ID**
   - Under **Project → Manage → Services & integrations**, make sure a
     **watsonx.ai Runtime** service instance is associated with the project
4. **Note your region URL** (shown in the watsonx.ai console / instance
   details), e.g. `https://us-south.ml.cloud.ibm.com` for Dallas.

### Step 5: Create your `.env` file
```bash
# Mac/Linux
cp .env.example .env

# Windows
copy .env.example .env
```
Open `.env` in any text editor and fill in your real IBM Cloud values:
```
WATSONX_API_KEY=your_ibm_cloud_api_key_here
WATSONX_PROJECT_ID=your_watsonx_project_id_here
WATSONX_URL=https://us-south.ml.cloud.ibm.com
GRANITE_MODEL_ID=ibm/granite-3-8b-instruct
```
Save the file.

### Step 6: Verify everything is set up correctly
```bash
python setup_check.py
```
This checks your Python version, confirms every package installed correctly,
and confirms your `.env` has valid IBM Cloud credentials. Fix anything it
flags before moving on.

### Step 7: Run the app
```bash
streamlit run app.py
```
It opens automatically at `http://localhost:8501` in your browser.

## Using the App

1. **Sidebar**: upload your resume PDF, paste the job description, optionally
   type a target company.
2. **Mock Interview tab**: pick interview type + difficulty → "Generate
   Interview Questions" (calls the IBM Granite model on watsonx.ai) →
   type your answer in the text box and click "Submit Answer" → get instant
   AI feedback per question.
3. **Skill Roadmap tab**: click "Analyze Skill Gaps" → see matched skills
   and a study plan for missing ones.

## Deploying on IBM Cloud (optional)

This app can be containerized and deployed on **IBM Cloud Code Engine** so
grading/demo reviewers can access it via a public URL:

```bash
# 1. Build a container image (Dockerfile not included by default — add one
#    that runs: streamlit run app.py --server.port 8080 --server.address 0.0.0.0)
ibmcloud cr build -t us.icr.io/<namespace>/interview-coach:latest .

# 2. Create (or select) a Code Engine project
ibmcloud ce project create --name interview-coach-project
ibmcloud ce project select --name interview-coach-project

# 3. Deploy the app, passing your IBM Cloud credentials as env vars/secrets
ibmcloud ce application create --name interview-coach \
  --image us.icr.io/<namespace>/interview-coach:latest \
  --env WATSONX_API_KEY=<your_key> \
  --env WATSONX_PROJECT_ID=<your_project_id> \
  --env WATSONX_URL=https://us-south.ml.cloud.ibm.com \
  --env GRANITE_MODEL_ID=ibm/granite-3-8b-instruct \
  --port 8080
```
For a class/internship submission, running locally with `streamlit run app.py`
and demoing via screen share/recording is usually sufficient — deployment is
an optional stretch goal.

## Notes for Report / Limitations

- Company-specific flavoring relies on the Granite model's training
  knowledge, not a live web lookup — stronger for large, well-known
  companies, more generic for small/recent ones.
- Text generation depends on your IBM Cloud plan's rate limits/quota; expect
  brief delays under heavy use on the Lite plan.
- Feedback parsing has a fallback path if the model doesn't return valid
  JSON on a given call, so the app won't hard-crash mid-demo.
- All AI/LLM logic is isolated in `ai_engine.py`, so the watsonx.ai project
  or Granite model ID can be swapped/upgraded without touching the
  Streamlit UI in `app.py`.
